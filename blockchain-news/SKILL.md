# 区块链新闻摘要

抓取三路数据源的当日新闻，整理成一份结构化的中文摘要报告。本 skill 自包含，
不依赖任何外部项目代码，只需要 Python 和 `requests` 库。

**职责边界**：本 skill 只负责抓取原始新闻并按规则整理输出报告，不发送邮件、
不推送到任何平台。分发由使用者自行决定。

## 数据源

`fetch_news.py` 会抓取以下三路数据：

1. **KOL 推文** — 区块链意见领袖的推文（作者、时间、正文、原文链接）。
2. **媒体快讯** — 加密媒体的即时新闻（来源、标题、时间、摘要、链接）。
3. **AI 日报** — 当日 AI 领域新闻（标题、链接）。

KOL 和媒体默认回溯 **12 小时**；AI 日报取当日。

## 使用步骤

### 1. 抓取原始数据

在 skill 目录下运行脚本，它会把三路数据以 JSON 打印到标准输出：

```bash
python fetch_news.py                    # 默认回溯 12 小时，AI 日报取今天
python fetch_news.py --hours 24         # 回溯 24 小时
python fetch_news.py --date 2026-07-05  # 指定 AI 日报日期
```

若缺少依赖，先安装：`pip install requests`。

输出 JSON 结构：

```json
{
  "kol":     [{"time": "...", "author": "...", "url": "...", "content": "..."}],
  "media":   [{"time": "...", "source": "...", "title": "...", "url": "...", "intro": "..."}],
  "ai_news": [{"title": "...", "link": "..."}],
  "errors":  {}
}
```

某一路抓取失败时，对应数组为空，并在 `errors` 里记录原因——不会中断其它数据源。

### 2. 按规则整理输出

拿到 JSON 后，按下面的格式整理成一份中文 Markdown 报告。报告分三个板块，
每个板块**保留原文链接**，措辞客观、排版清晰，结尾不做总体总结、不加多余对话。

#### 板块一：区块链 KOL 新闻

阅读 `kol` 中的推文，提炼每位 KOL 的核心观点，逐条列出：

```
## 区块链 KOL 新闻

消息1：[作者名] 在 [日期] 提到 [观点摘要]。
来源：[原文链接]

消息2：...
```

#### 板块二：区块链媒体新闻

阅读 `media`，按内容性质**分类**（技术发展 / 市场动态 / 政策法规 等），
每类下列出代表性新闻：

```
## 区块链媒体新闻

### 市场动态
1. [标题](链接)。内容摘要。
2. ...

### 技术发展
1. [标题](链接)。内容摘要。

### 政策法规
1. [标题](链接)。内容摘要。
```

分类依据新闻内容本身，只保留有实质信息的条目。

#### 板块三：AI 新闻

`ai_news` 只有标题和链接，直接列成清单，无需摘要：

```
## AI 新闻

1. [标题](链接)
2. [标题](链接)
```

### 3. 报告整体格式

- 顶部标题：`# YYYY-MM-DD 区块链新闻`（日期为当天）。
- 三个板块依次为：区块链 KOL 新闻 → 区块链媒体新闻 → AI 新闻。
- 全程使用中文，用 Markdown 语法，注意排版美观。
- 每条新闻都要带原文链接，方便溯源。
- 结尾不写总结语、不追加对话。

## 注意事项

- 数据源为第三方公开接口，接口地址或返回结构可能变动；某板块突然为空多半是
  接口变化，而非报告逻辑出错，可查看脚本 stderr 的 `[warn]` 提示。
- 抓取本身不去重，跨次运行可能出现重复条目，整理时可自行合并明显重复的新闻。
- 调整回溯窗口用 `--hours`；如需更多数据源，可在 `fetch_news.py` 中新增
  `fetch_*` 函数并把结果并入输出 JSON。
